# tackle-backlog

BACKLOG.md の未完了タスクを選択し、プランモードで設計・実装・完了まで一貫して進める。

## ワークフロー

### Step 1: BACKLOG.md 読み込み・タスク一覧表示

`BACKLOG.md` を Read ツールで読み込み、`- [ ]` の未完了タスクを番号付きで列挙する。

```
未完了タスク一覧:
  1. commit スキルに自動コミット内容チェック機能を追加
     概要: コミット前にステージ対象ファイルの内容をテキスト提示する手順を明示
  2. create-git-wiki SKILL.md（377行）のスリム化検討
     概要: スクリプト化できるロジックを読んで確認
  ...
```

- `- [x]` の完了済みタスクはスキップする
- タスクがゼロなら「未完了タスクはありません」と報告して終了する

### Step 2: 実施タスクの選択

- 引数付きで呼ばれた場合（例: `/tackle-backlog 2`）は引数の番号をそのまま使う
- 引数がない場合は `AskUserQuestion` でユーザーに番号を選ばせる

```
どのタスクを進めますか？ 番号を入力してください（例: 1）
```

### Step 3: タスク分析 → プランモード

選択タスクの概要・背景・参照先を表示してから `EnterPlanMode` を呼び、実装プランを作成する。

- 参照先ファイルがあれば Read ツールで読み込んで分析に含める
- プランには変更ファイル・実装手順・確認方法を含める

### Step 4: 実装

プラン承認後に実装を実行する。既存のプランワークフローに委譲する。

### Step 5: BACKLOG.md 更新 + コミット

実装完了後に以下を行う:

1. **BACKLOG.md を更新する**

   `- [ ]` を `- [x]` に変更し、完了情報を追記する:

   ```
   - [x] **タスクタイトル**
     - 概要・背景
     - 対応: commit `{ハッシュ}` ({YYYY-MM-DD})
   ```

2. **`/commit` スキルを呼び出してコミットする**

   コミットメッセージ例:
   ```
   fix(backlog): complete task "{タスクタイトル}"
   ```

## 注意事項

- BACKLOG.md がリポジトリルートに存在しない場合はユーザーに確認して終了する
- タスクの実装中に別タスク・改善案が出てきたら BACKLOG.md に追記し、現タスクを中断しない
- プランモードへの遷移前にユーザーが「やめる」と言った場合は即座に終了する
