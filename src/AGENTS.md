# グローバルエージェント設定

## 基本方針

- 応答・ドキュメント記述は日本語
- コード・スクリプトで解決できるものをプロンプトより優先する。CLI/シェルスクリプトで解決できるものは MCP より優先する
- プロジェクトのランタイム管理・タスクランナー・環境変数設定には mise を使用する

## バックログ管理

- セッション中に本題と別件のタスク・改善案が出てきたら `BACKLOG.md` に追記する
- 形式: `- [ ] **タイトル**\n  - 概要・背景・参照先`
- 実装・対応は別セッションで行う。現セッションの作業を中断しない

## ルール参照

Git・シェルスクリプト・フックに関するルールは、作業内容に応じて以下を適宜参照すること:

- `rules/git.md` — Git ワークフロールール
- `rules/shell.md` — シェル/PowerShell ベストプラクティス
- `rules/hook.md` — フックのテスト方法・ライフサイクル
